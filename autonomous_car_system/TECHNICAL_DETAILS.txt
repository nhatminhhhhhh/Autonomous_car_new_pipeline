===============================================================================
  AUTONOMOUS CAR SYSTEM — Technical Details, Algorithms & Pipeline
===============================================================================

1. SYSTEM OVERVIEW
--------------------------------------------------------------------------------
The autonomous car system uses a camera-based perception-to-control pipeline
running on a Jetson Nano (with Windows fallback for development). It uses
segmentation-mask-based steering:

   Camera Frame
       |
       v
   [RoadSegNet]  ->  road segmentation mask (3 classes: background, road, lane)
       |
       v
   [MaskDrivingNet]  ->  steering angle prediction (-1.0 to +1.0)
       |
       v
   [ESP32Controller]  ->  servo motor commands via USB UART

The system supports three operating modes:
  - test:        Real-time road segmentation visualization only
  - collect:     Teleoperation data collection (WASD control, saves masks)
  - autonomous:  Full end-to-end autonomous driving

3. MODELS
--------------------------------------------------------------------------------

3a. RoadSegNet (road_segnet.py)
  - Architecture: Lightweight encoder-decoder (U-Net variant)
  - Encoder: 4 stages of SeparableDoubleConv + MaxPool2d (2x)
      features = [8, 16, 32, 64]  (channel depths)
  - Bottleneck: SeparableDoubleConv(64, 128)
  - Decoder: 4 stages of Upsample(bilinear, 2x) + skip connections + conv
  - Output: 1x1 Conv2d(features[0], num_classes)  ->  pixel-wise logits
  - Key design choices:
      * SeparableConv2d (depthwise + pointwise) reduces parameters vs standard conv
      * Dropout2d(0.2) after each decoder block for regularization
      * Input: 128x128 RGB, Output: 128x128 class indices (3 channels)
  - Parameter count: ~95K (very lightweight for embedded deployment)

3b. MaskDrivingNet (mask_driving_net.py)
  - Architecture: Tiny CNN for steering regression from segmentation mask
  - Input: 1-channel mask (class index / num_classes), 128x128
  - Output: single float ∈ [-1.0, 1.0] (steering angle)
  - Encoder: 4 Conv blocks (stride=2) downsample 128->64->32->16->8
      channels: 1->16->32->64->128
  - Head: Global Average Pooling -> Linear(128,64) -> Dropout(0.3) -> Linear(64,1) -> Tanh
  - Parameter count: ~80K


4. TRAINING PIPELINE
--------------------------------------------------------------------------------

4a. RoadSegNet Training (train_road.py)
  - Loss: CrossEntropyLoss (pixel-wise classification)
  - Optimizer: Adam (lr=5e-3, weight_decay=5e-5)
  - Scheduler: ReduceLROnPlateau (factor=0.5, patience=10)
  - Early stopping: patience=15 epochs on validation loss
  - Batch size: 16
  - Image size: 128x128
  - Data augmentation (via albumentations):
      * HorizontalFlip (p=0.5)
      * RandomBrightnessContrast (p=0.3)
      * GaussNoise (p=0.2)
      * Affine scale/translate (p=0.3)
  - Dataset: my_dataset/{train,val}/{images,labels}/
  - Checkpoints saved every 10 epochs + best model by val loss

4b. MaskDrivingNet Training (train_mask_driving.py)
  - Loss: HuberLoss(delta=0.1) — robust to steering outliers
  - Optimizer: Adam (lr=1e-3, weight_decay=1e-4)
  - Scheduler: ReduceLROnPlateau (factor=0.5, patience=5)
  - Early stopping: patience=15 epochs
  - Batch size: 32
  - Gradient clipping: max_norm=1.0
  - Dataset: collected_driving/ (mask_*.png + frame_*.json) — masks saved during teleoperation
  - Augmentation: random horizontal flip with steering sign inversion
  - Edge: training does NOT depend on RoadSegNet; masks are pre-computed

4c. TensorRT Export (export_trt.py)
  - Converts RoadSegNet to TensorRT FP16 engine via torch2trt
  - Saves as best_model_trt.pth alongside the PyTorch checkpoint
  - Runtime: RoadDetector checks for _trt.pth suffix and loads TRT if available


5. INFERENCE PIPELINE
--------------------------------------------------------------------------------

5a. RoadDetector (inference/road_detector.py)
  - Loads RoadSegNet (PyTorch or TensorRT)
  - Preprocessing: resize to 128x128, RGB conversion, normalize (ImageNet stats)
  - Inference: single forward pass, argmax over classes
  - Postprocessing: nearest-neighbor resize back to original frame size
  - Color LUT: [0,0,0] = background, [0,200,0] = road (green), [0,220,220] = lane (cyan)
  - FP16 inference when CUDA available

5b. MaskSteeringPredictor (inference/mask_steering.py)
  - Loads MaskDrivingNet (PyTorch only)
  - Preprocessing: resize mask to 128x128, normalize to [0, 1]
  - Output: single float clamped to [-1.0, 1.0]
  - If model is None (no checkpoint found), returns 0.0 (straight)

5c. AutonomousPipeline (inference/pipeline.py)
  - Combines RoadDetector + MaskSteeringPredictor + CameraThread
  - Main loop: reads frames, runs RoadSegNet -> mask -> MaskDrivingNet -> steering
  - FPS counter for performance monitoring


6. HARDWARE INTERFACE
--------------------------------------------------------------------------------

6a. Camera (hardware/camera.py)
  - Platform detection via /etc/nv_tegra_release and /proc/device-tree/model
  - Jetson Nano: IMX219 camera via GStreamer pipeline (1640x1232 capture, 640x480 display)
      "nvarguscamerasrc ! video/x-raw(memory:NVMM) ! nvvidconv ! videoconvert ! appsink"
  - Windows: DirectShow (DSHOW) or Media Foundation (MSMF) backends
  - CameraThread: background thread with bounded queue (maxsize=2), drops stale frames

6b. ESP32 Controller (hardware/esp32_controller.py)
  - Serial communication with ESP32 over USB UART
  - Auto-detects port by known VID/PID (CP2102, CH340, FT232, etc.)
  - Sends commands in format "S<steering>,T<throttle>\n"
  - Hardware reset via DTR/RTS, simulation fallback if no port found


7. DATA COLLECTION
--------------------------------------------------------------------------------
  - TeleoperationCollector (data/teleoperation.py):
      * WASD keyboard control (pynput listener)
      * A/D = steering, W/S = throttle, Space = brake, R = toggle recording
      * Runs RoadSegNet in real-time to generate segmentation mask
      * Saves (mask_XXXXXX.png, frame_XXXXXX.json) with steering + throttle
      * Falls back to saving raw images if no RoadSegNet checkpoint provided
  - PrepareDataset (data/prepare_dataset.py):
      * Converts LabelMe JSON polygon annotations to PNG segmentation masks
      * Supports multiple label classes via labels.txt
      * Splits data into train/val (80/20)
      * Output: {train,val}/{images,labels}/


8. CONFIGURATION (configs/config.py)
--------------------------------------------------------------------------------
  - device:            CUDA if available, else CPU
  - RoadSegNet:        3 classes, 128x128 input, 16 batch, 200 epochs, lr=5e-3
  - MaskDrivingNet:    1-channel 128x128 input, 32 batch, 80 epochs, lr=1e-3
  - Camera:            640x480, 60fps
  - ESP32:             Serial UART @ 115200 baud
  - Paths:             checkpoints/ (road_seg/, mask_driving/), data/collected_driving/
